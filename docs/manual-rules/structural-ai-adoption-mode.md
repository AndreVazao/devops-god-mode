# Structural AI Adoption Mode

## Decisão

O DevOps God Mode adota **modo B** na fase atual:

- IA estrutural híbrida;
- motor principal grátis/local/regra-based;
- integração opcional com APIs externas quando necessário;
- evolução futura para **modo C** quando os programas e websites estabilizarem e começarem a gerar receita suficiente.

## Interpretação prática

### Agora
O sistema deve privilegiar:
- heurísticas estruturais;
- scoring;
- detecção de padrões;
- graph reasoning;
- regras persistentes;
- memória em Supabase;
- inferência local sem custo obrigatório.

### Depois
Quando houver orçamento operacional estável, o sistema pode ativar:
- LLM structural reasoning;
- análise arquitetural mais profunda;
- explicações inteligentes multi-repo;
- propostas avançadas de refactor e consolidação.

## Regra de produto

O God Mode deve nascer funcional e útil **sem depender de APIs pagas**.

Integrações pagas ficam como:
- opcionais;
- plugáveis;
- desligadas por defeito até decisão explícita do owner.

## Objetivo

Garantir:
- custo controlado no arranque;
- crescimento sem bloqueio financeiro;
- caminho claro para inteligência premium futura.
