# Provider onboarding cockpit

## Routes

- `/api/provider-onboarding-cockpit/dashboard`
- `/app/provider-onboarding`

## Objective

Give one place to inspect provider readiness, first-run login steps, and multirepo planning before starting a new project.

## What it shows

- requested providers for the target project
- live capability visibility for each provider
- blockers when providers are not ready enough
- first-run login and session capture steps
- resulting multi-repo plan for frontend and backend
