# God Mode remediation cockpit

## Routes

- `/api/godmode-remediation/plan`
- `/app/godmode-remediation`

## Objective

Transform diagnostics into ordered next actions so the operator can stop guessing and start clearing blockers faster.

## What it prioritizes

- missing providers
- missing secrets in vault
- pending operator threads
- recent failures and restart-required events
- backlog readiness when no critical blockers remain

## Why this matters

The diagnostics cockpit tells what is wrong. The remediation cockpit tells what to do next, in priority order, so the God Mode can move faster towards helping with the backlog of pending projects.
