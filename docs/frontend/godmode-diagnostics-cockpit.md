# God Mode diagnostics cockpit

## Route

- `/app/godmode-diagnostics`

## Objective

Give a direct place to inspect readiness, blockers, recent failures and recovery signals so project creation can move faster with less guesswork.

## What this cockpit shows

- pending thread attention
- provider readiness
- vault readiness
- tenant scope readiness
- recent failure events
- recent recovery events
- recent offline queue activity
- blocker summary

## Why this matters

When many projects are waiting, the main problem is usually not lack of ideas but lack of visibility over what is broken, incomplete or inconsistent. This cockpit compresses that visibility into one operational screen.
