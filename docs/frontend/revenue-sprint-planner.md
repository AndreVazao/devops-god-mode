# Revenue Sprint Planner

## Routes

- `/api/revenue-sprint/status`
- `/api/revenue-sprint/package`
- `/api/revenue-sprint/dashboard`
- `/api/revenue-sprint/create`
- `/api/revenue-sprint/approval-card`
- `/api/revenue-sprint/sprints`
- `/app/revenue-sprint`

## Objective

Turn the Monetization Readiness Matrix into short, approved execution sprints.

## Why it matters

The matrix says what can make money. The sprint planner turns that into 5-task execution packs per priority project, with approval cards and AndreOS history.

## Flow

1. Generate monetization matrix.
2. Select the top critical/high projects.
3. Create tasks for repo, memory, MVP, offer and delivery checklist.
4. Create a mobile approval card.
5. Execute only after operator approval.

## Safety

Revenue Sprint Planner does not execute destructive work. It creates an approval-gated plan.
